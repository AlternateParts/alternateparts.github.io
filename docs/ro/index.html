    Route Optimization (RO) APIs

    Welcome to Route Optimization (RO) API docs. Use RO APIs to create, manage and optimize the routes for your pickup & delivery operations.

    With these APIs, you can

    • Sign up for AlternateParts
    • Login and obtain your RO API key to access these APIs
    • Create and read organizations
    • Create and read tasks
    • Create and read drivers
    • Organize all the drivers, tasks and optimizations operations under an organization

    How it works?

    An overview of how everything is related in Route Optimization service

    • You create your account by Signing up

    • You add one or more Organization to your account. If you deliver orders to customers of your clients, each Organization can represent one of your clients. It is a way of organizing related operations under one umbrella. If you only deliver your own organization's orders, you can just create a single organization

    • You can add Drivers to each Organization. These drivers will be responsible for delivery the tasks to specified locations after Route Optimization is done

    • Each day you deliver multiple orders to your customers or customers of your clients. Each of these delivery orders / shipment task are called Tasks. Every task has a location where it must be delivered

    How to use API

    • Obtain API key by signing up for AlternateParts

    • Every API endpoint starts with

    https://api.alternate.parts/vrp/api/v1/

    • Include the API key in header of every request as

    Authorization: <API_KEY>

    • Add organizations

    • Add drivers to your organzation

    • Add tasks that you want to deliver at specified locations

    • Call Route Optimization endpoint to assign tasks to drivers in an efficient way where each driver has to cover minimum distance

    Important notes

    • At the moment, Route Optimization API only optimize for distance. Support for vehicle capacity optimization and delivery time optimization will be available form next release

    • <OBJECT_ID>, <ORG_ID>, <TASK_ID>, <DRIVER_ID> are strings IDs that are unique for every object (row) in every class (table)

    • <API_KEY> is also a string

    • Use your API keys from the server or cloud workers. Do not use API keys on the front-ends where there is a chance that someone can find it. This means you must run your code on the server or a dedicated worker computer to call RO APIs

    1. Check API Status

    A test call to root API endpoint

    curl "https://api.alternate.parts/vrp/api/v1/" \
  -X POST

    import requests

url = "https://api.alternate.parts/vrp/api/v1/"

r = requests.post(url)

    Response

    {
  "api": "v1",
  "vrp": "v0.1"
}

    To check whether RO APIs are up and running, make a POST request to the root API endpoint:

    https://api.alternate.parts/vrp/api/v1/

    "api": "v1" in the response means all good.

    HTTP Request

    POST https://api.alternate.parts/vrp/api/v1/

    2. Sign up

    Provide your email and a strong password

    curl "https://api.alternate.parts/vrp/api/v1/signup" \
  -X POST
  -H "Content-Type: application/json"
  -d '{"email": "<EMAIL>", "password": "<PASSWORD>"}'

    import requests

url = "https://api.alternate.parts/vrp/api/v1/signup"
headers = {
  "Content-Type": "application/json", 
  }
data = {
  "email": "<EMAIL>", 
  "password": "<PASSWORD>"
  }

r = requests.post(url, headers=headers, data=data)

    Response contains id of newly added account

    {
  "objectId": "<OBJECT_ID>"
}

    Use your company email to sign up for AlternateParts to use RO APIs. Also provide a strong password.

    It's important that you use an active email address because we will use this email to communicate with you important things.

    HTTP Request

    POST https://api.alternate.parts/vrp/api/v1/signup

    URL Parameters

    Parameter Required Description
    email true your company email
    password true a strong password

    3. Obtain API key

    Use your account's email and password to obtain your API key

    curl "https://api.alternate.parts/vrp/api/v1/login" \
  -X POST
  -H "Content-Type: application/json" 
  -d '{"email": "<EMAIL>", "password": "<PASSWORD>"}'

    import requests

url = "https://api.alternate.parts/vrp/api/v1/login"
headers = {
  "Content-Type": "application/json", 
}
data = {
  "email": "<EMAIL>", 
  "password": "<PASSWORD>"
}

r = requests.post(url, headers=headers, data=data)

    Response contains your API key and account ID

    {
  "apiKey": "<API_KEY>",
  "objectId": "<OBJECT_ID>"
}

    The Login API endpoint returns the API key associated with your account.

    HTTP Request

    POST https://api.alternate.parts/vrp/api/v1/login

    URL Parameters

    Parameter Required Description
    email true your account's email
    password true your password

    4. Organizations

    Organize your operations under an Organization. Creating organization is as simple as providing name and description to the right API endpoint.

    Creating multiple organizations is a good way to organize tasks and drivers if you have different drivers for each organization. If you provide delivery services to multiple different clients, create multiple organizations, each represening a single client.

    4.1. Create an Organization

    Provide name and description about your organization

    curl "https://api.alternate.parts/vrp/api/v1/orgs" \
  -X POST
  -H "Authorization: <API_KEY>"
  -H "Content-Type: application/json" 
  -d '{"name": "<NAME>", "description": "<DESCRIPTION>"}'

    import requests

url = "https://api.alternate.parts/vrp/api/v1/orgs"
headers = {
  "Authorization": "<API_KEY>",
  "Content-Type": "application/json", 
}
data = {
  "name": "<NAME>", 
  "description": "<DESCRIPTION>"
}

r = requests.post(url, headers=headers, data=data)

    Response contains ID of newly created organization

    {
  "objectId": "<OBJECT_ID>"
}

    Create a single organization. To create multiple organizations, create them one by one.

    HTTP Request

    POST https://api.alternate.parts/vrp/api/v1/orgs

    URL Parameters

    Parameter Required Description
    name true name of organization
    description true details of organization

    5. Tasks

    Tasks are the delivery reponsibility units that are assigned to each driver on a given duty. They represent the individual orders that you have to deliver to the customers.

    5.1. Create a Task

    Provide organization ID, name and pin location of the task

    curl "https://api.alternate.parts/vrp/api/v1/tasks" \
  -X POST
  -H "Authorization: <API_KEY>"
  -H "Content-Type: application/json" 
  -d '{"orgId": "<ORG_ID>", "name": "<NAME>", "location": [<LATITUDE>, <LONGITUDE>]}'

    import requests

url = "https://api.alternate.parts/vrp/api/v1/tasks"
headers = {
  "Authorization": "<API_KEY>",
  "Content-Type": "application/json", 
  }
data = {
  "orgId": "<ORG_ID>", 
  "name": "<NAME>", 
  "location": [<LATITUDE>, <LONGITUDE>]
  }

r = requests.post(url, headers=headers, data=data)

    Response contains ID of newly created task

    {
  "objectId": "<OBJECT_ID>"
}

    Each task has a location where it must be delivered. And every task belongs to an organization.

    HTTP Request

    POST https://api.alternate.parts/vrp/api/v1/tasks

    URL Parameters

    Parameter Required Description
    orgId true ID of organization to which this task belongs
    name true name of the task
    location true array containing latitude and longitude of where to drop the task

    6. Drivers

    A driver is responsible for delivering assigned tasks to the specified locations.

    You can think of drivers as delivery vehicles as well if you Optimize Route based on vehicle types.

    Location of the driver could be his/her residence or start location that usually remains the same. You can also set it to equal to the location of your warehouse.

    6.1. Create a Driver

    Provide organization ID, name and location of driver

    curl "https://api.alternate.parts/vrp/api/v1/drivers" \
  -X POST
  -H "Authorization: <API_KEY>"
  -H "Content-Type: application/json" 
  -d '{"orgId": "<ORG_ID>", "name": "<NAME>", "location": [<LATITUDE>, <LONGITUDE>]}'

    import requests

url = "https://api.alternate.parts/vrp/api/v1/drivers"
headers = {
  "Authorization": "<API_KEY>",
  "Content-Type": "application/json", 
  }
data = {
  "orgId": "<ORG_ID>", 
  "name": "<NAME>", 
  "location": [<LATITUDE>, <LONGITUDE>]
  }

r = requests.post(url, headers=headers, data=data)

    Response contains ID of newly created driver

    {
  "objectId": "<OBJECT_ID>"
}

    A driver belongs to an organization and while Route Optimization operation, they can only be assigned tasks within their organization.

    HTTP Request

    POST https://api.alternate.parts/vrp/api/v1/drivers

    URL Parameters

    Parameter Required Description
    orgId true ID of organization to which this driver belongs
    name true name of the driver
    location true array containing latitude and longitude of driver's daily starting location

    7. Optimize Route

    Optimize the Route for provided tasks and provided number of drivers

    curl "https://api.alternate.parts/vrp/api/v1/optimize" \
  -X POST
  -H "Authorization: <API_KEY>"
  -H "Content-Type: application/json" 
  -d '{"numDrivers": <INTEGER>, "taskIds": [<TASK_IDs>]}'

    import requests

url = "https://api.alternate.parts/vrp/api/v1/optimize"
headers = {
  "Authorization": "<API_KEY>",
  "Content-Type": "application/json", 
  }
data = {
  "numDrivers": <INTEGER>, 
  "taskIds": [<TASK_IDs>]
  }

r = requests.post(url, headers=headers, data=data)

    Response contains tasks belonging to provided orgId

    {
  "route": {
    // route information. tasks <> drivers mapping
  }
}

    Optimize Route for provided tasks.

    HTTP Request

    POST https://api.alternate.parts/vrp/api/v1/optimize

    URL Parameters

    Parameter Required Description
    numDrivers true Number of drivers who will be part of this delivery operation
    taskIds true array containing the IDs of tasks that must be routed and optimized
